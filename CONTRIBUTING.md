# Contributing to DSS

Thank you for your interest in contributing to the DSS project!

## How to Contribute

1. Fork this repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Test locally:
  ```bash
  npm install
  npm run dev
  ```
5. Commit your changes with clear messages
6. Push to your branch (`git push origin feature/amazing-feature`)
7. Open a Pull Request

## Pull Request Process

- Describe what your changes accomplish
- Reference any related issues
- Ensure all tests pass and code follows the project's style
- PRs are reviewed by maintainers
- Approved PRs are merged to this repository
- Changes are then automatically synced back to the main monorepo
- You'll be credited in both repositories!

## Code Standards

- Follow existing code style
- Add tests for new features where applicable
- Update documentation as needed
- Ensure TypeScript types are correct

## Questions?

- Open an issue with your question
- Check existing issues for similar questions
- Reach out to the maintainers

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.

## Licensing
